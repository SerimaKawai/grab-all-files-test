セッション破壊リンク（クロールが踏んではいけない）

popup.js の crawlUrl 内 isSessionMutatingLink(parsed, text) は、 認証付き GET（credentials:"include"）で踏むとユーザーをサインアウトさせてしまうリンクを クロール対象から除外します（Coursebase の Angular SPA、Liferay の /c/portal/logout で報告）。 サインアウトすると取得済みのダウンロード URL まで失効し、再読み込みまでファイルが取れなくなるため、 これらは結果一覧に出してはならず、ディープスキャンでクロールしてもいけません。
判定は 2 系統。
(1) URL の pathname + search（小文字化）に対する正規表現 /(?:^|[/?#&=._-])(?:log|sign)[-_]?(?:out|off)(?:$|[/?#&=.-])/ ＝ logout / signout / logoff / signoff および log-out / sign_out などの区切り変種、 ?action=signout のようなクエリ値も対象。
(2) 可視リンクテキストに対する log out / sign out / log off / sign off（空白・ハイフン任意）と ログアウト / サインアウト / ログオフ / サインオフ。
  • ログアウト
  • Sign out（/account/sign_out）
  • Liferay ログアウト（/c/portal/logout）
  • Sign out（?action=signout クエリ）
  • sign-out（ハイフン変種）
  • logoff（log off 変種）
  • サインアウト
  • ログオフ
最後の 2 件はパスに logout 系の語がなくても、可視テキスト（サインアウト / ログオフ）でテキスト判定に引っかかり除外される例。逆に /blog/out-and-about や /design-out のような無関係 URL は境界条件により誤マッチしない想定です。

期待される検出

① 結果一覧に出てはならない（全リンク共通）: 本ページの全リンク。スキャン結果（ファイル一覧）に 1 件も現れないこと。
・SourceForge 一覧/レビュー/管理・naturalearthdata /downloads/<slug>/・「download」を含むナビ URL ＝ ファイル signal が無い（urlMatch/urlPattern false）ためファイルにならない。
・chrome:// / about: ＝ スキャン不可スキーム。
② さらにクロール（フロンティア enqueue）もしてはならない: 「セッション破壊リンク」節の 8 件＝isSessionMutatingLink によりクロール enqueue から除外。 chrome:// / about: も同様にクロール対象外。 これらがディープスキャンでページキューに入ったら isSessionMutatingLink（またはスキームガード）のリグレッションです。
※ 補足: SourceForge /files/ や naturalearthdata /downloads/<slug>/ は nonFileUrlRe でブロックされるわけではないため、ディープスキャンでは 同一サイトのナビページとして enqueue され得ます（これは正しいクローラ挙動）。 保証されるのは「ファイル結果に出ないこと」であって、これら 4 件は必ずしも「クロールしない」対象ではありません。
検出されるべき: なし（このページは誤検出トラップ専用で、ダウンロード対象の実ファイルは置いていません）。
