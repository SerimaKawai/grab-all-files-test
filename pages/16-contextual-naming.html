D. PDF hex /Title デコード

対象: lib/popup-utils.js の decodePdfHexText（Shift_JIS / UTF-16 / UTF-8 を 判定してデコード）と tidyPdfDocTitle（「Microsoft Word - 」接頭辞＋元ドキュメント拡張子を除去）。 呼び出しは popup.js の fetchOneMeta 内 safeTitleStr()。
壊れた Office→PDF 書き出しでは /Title が「角括弧つき16進文字列」そのもの （例 <4D6963726F736F667420576F7264202D20…>）として保存されることがあり、 pdf-lib / pdf.js はこれをそのまま ASCII 文字列で返すため、デコードしないと そのまま <4D69…> がタイトルとして混入する。
  • PDF をダウンロード
このファイルの /Title は、文字列リテラル <FEFF004D006900630072006F0073006F0066007400200057006F007200640020002D002056FD8A9E002E0064006F00630078> を保持している。これは UTF-16BE（先頭 FEFF = BOM）で 「Microsoft Word - 国語.docx」をエンコードした16進バイト列。
decodePdfHexText が BOM を見て UTF-16BE としてデコード → Microsoft Word - 国語.docx、続いて tidyPdfDocTitle が 「Microsoft Word - 」接頭辞と「.docx」拡張子を除去 → 国語 になる。

E. 列ヘッダ科目（種別は列、行ではない）

対象: lib/popup-utils.js の gafColumnSubjectWord（リンクのセルを覆う 列ヘッダ th の文字列を読み、GAF_SUBJECT_WORDS に一致すれば科目として採用。 セル単位で都度評価＝キャッシュしない）。CASE A は科目が行セルにあるのに対し、 ここでは科目が列ヘッダにある対照ケース。
各列の科目（英語 / 数学 / 理科）は列見出しに置き、行見出し（th）に種別（問題 / 解答）を置く。 composeContextualName は rowSubj が無いとき gafColumnSubjectWord(el) で列ヘッダ科目を補完し、 種別は行内の [問題PDF]/[解答PDF] アンカー（gafBlockHasTypeAnchor）で発火する。

F. URL からの科目/種別/年 復元

対象: lib/popup-utils.js の gafUrlExamTokens（URL パスの romaji exam slug kokugo=国語 / mondai=問題 等から科目・種別・年を復元。 同一ステムが科目と種別を両方含むとき(paired)のみ採用＝kokugo-mondai）。 呼び出しは composeContextualName で、行に種別ラベル [問題PDF]/[解答PDF]（gafBlockHasTypeAnchor）があり、かつ h3 から年が取れて種別ゲートを通過したあとに、URL トークンが科目を補完する。

重要（URL は単独では絶対に発火しない）: gafUrlExamTokens が返す科目/種別/年は、DOM 側の種別シグナル （ラベルの 問題/解答 = isType、補強された列ヘッダ種別 = effHeaderType、 または行内の種別アンカー = gafBlockHasTypeAnchor）が先に種別ゲートを通過した 場合にのみ補強(enrich)として使われる。
composeContextualName では if (!isType && !effHeaderType && !hasTypeAnchor) return ""; が URL 評価の前に種別ゲートとして効くため、種別の DOM 証拠が無い純粋な ../files/suugaku-mondai.pdf 等は、たとえ paired slug を持っていても合成されない。 これは日本語教育サイト全域での過剰発火を避けるための設計（adversarial review で URL 単独発火は3度撤去）。
