@font-face src — 検出されない

このページの head スタイルで sample.woff2 / sample.woff / sample.ttf を @font-face の src で参照していますが、 getComputedStyle() は @font-face の src を返さず、 cssRules 走査も無いため、これらフォントは 検出されません。 （フォントを検出させたい場合は <a href> など別経路が必要です。）

cursor / ::before content — 検出されない

この上にカーソル → sample.png    ::before に content: url(../files/sample.png)

cursor: url() と ::before/::after の content: url() は 読み取り対象外なので、これら経由の URL は検出されません。 なお sample.png 自体は上の background-image 系で別途検出されます （この URL が「検出される」のは ::before content 経由ではない点に注意）。

::before content の種別ラベルをファイル名へ復元

popup.js 内の gafPseudoText() は、アンカー自身に テキストノードが無いとき getComputedStyle(el, "::before") / ::after の content から可視キャプションを復元します。
resemom の公立高入試一覧は「問題 / 正答」ボタンを空アンカー＋CSS擬似要素 （.que::before{content:"問題"} 等）で描いており、textContent は空のため種別ゲートが発火せず命名が空振りしていました。復元した種別語は GAF_TYPE_WORDS（問題 / 正答 / 正答例 / 解答 …）に照合され、 composeContextualName() が周囲の学校・年度・科目と合成して 学校_年度_科目_種別 形式のファイル名を作ります。
重要: この復元はライブ DOM（実ページ）でのみ動作します。 クロール（DOMParser 複製）側にはスタイルシートも defaultView も無いため getComputedStyle が使えず、 本機能は完全に不活性です。「このページのみ」スキャンで確認してください。
