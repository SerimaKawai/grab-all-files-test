13. 直リンク動画 / 動画・音声要素 / SVG / 短縮 data-*

← トップ
2026-06-09 改訂。v4.5.x で「直リンク動画」が検出対象に追加されました。 検出ロジックは lib/file-utils.js の knownExtRe（mp4/m4v/mov/webm/avi/mkv/wmv/flv/f4v/mpeg/mpg/m2v/3gp/3g2/ogv/divx/asf/rm/rmvb/vob を収録）と、 popup.js の scanFunc addUrl / extInPath、表示ゲート shouldKeepScannedItem（scanTypeGroups の video グループ）にあります。
重要な仕様（要確認）: 動画はあくまで <a href="x.mp4"> や URL 末尾が動画拡張子の直リンクのみ検出されます。 popup.js の deepQuery("video, audio") ループは poster（画像）と <track>（字幕）だけを収集し、 <video src> や <source src> 自体、 <audio> の音声本体は収集しません。 ストリーミング（HLS .m3u8 / DASH .mpd / .m4s / .m2ts / .mts）と 全音声、blob/傍受メディアは avExtRe（lib/file-utils.js / content.js ネットワーク経路ガード / page-interceptor.js）で恒久ブロックされます。
※ .ts（MPEG-TS セグメント）は否定テストに使えません。ts は TypeScript と拡張子が衝突し、web スキャングループ（デフォルト ON）の対象なので直リンクは検出されます。ストリーミングの否定確認には avExtRe が確実にブロックする .m4s を使います。
期待検出ユニーク URL: 16 件（同一ファイルを複数経路で参照しているので重複排除後の件数）。

① 直リンク動画（v4.5.x で対応） — 検出される

<a href="x.mp4"> 形式の直リンク、または URL 末尾が動画拡張子のもの。 これらは knownExtRe にマッチし、video スキャングループ（デフォルト ON）で表示されます。 すべて検出されます。

  • 講義録画 (sample.mp4)
  • プロモ動画 (sample.mov)
  • アーカイブ動画 (sample.mkv)
  • Web 動画 (sample.webm)
  • 旧形式動画 (sample.avi)
  • Windows Media (sample.wmv)
  • Flash 動画 (sample.flv)
  • iTunes 動画 (sample.m4v)
  • MPEG 動画 (sample.mpeg)
  • モバイル動画 (sample.3gp)

② <video> 要素（src / source / track / poster）

仕様: <video src> と各 <source src> は 収集されません（動画本体は直リンクのみ対象）。 収集されるのは poster（画像）と <track>（字幕）だけです。

  • video + 複数 source（mp4 / webm / ogv）（source は検出されない） + track 字幕（検出される） + poster（検出される）:

注: sample.mp4 / sample.webm は①の直リンクで既に検出済みです。 ②の <source> 経路では新たに検出されません。sample.ogv は直リンクが無いため （<source> でしか参照されない）検出されません。

③ <audio> 要素（src / source 5 MIME） — すべて検出されない

仕様: <audio> 要素は本体 src も <source> も収集しません。 poster も付きません（poster は video 専用）。音声拡張子は knownExtRe に無く、 avExtRe でもブロックされるため一切検出されません。

  • audio[src] 直接参照（検出されない）:
  • audio + 複数 source（mp3 / ogg / wav / m4a / flac）（すべて検出されない）:

④ SVG <image> / <use> 外部参照

外部 SVG への参照（modern href と legacy xlink:href 両方）。 <use href="#inline-sprite"> は同一ドキュメント内 fragment なので検出されません。

⑤ 画像系 data-* 短縮形（10種） — すべて検出される

popup.js の data-* 画像ローダー（addImg）が拾います。値の実拡張子で type hint を付与します。

  • data-poster
  • data-pic
  • data-photo
  • data-icon
  • data-cover
  • data-cover-image
  • data-srcset
  • data-banner
  • data-hero
  • data-avatar

⑥ メディア系 data-*（3種）

仕様: data-mp4 / data-webm は値が動画拡張子で終わるため extInPath で 検出されます（video グループ）。 data-mp3 は knownExtRe に無く avExtRe でもブロックされるため 検出されません。

  • data-mp4（検出される）
  • data-webm（検出される）
  • data-mp3（検出されない）

⑦ 検出されない動画 / 音声（avExtRe による恒久ブロック）

以下は直リンク <a href> として置いても、avExtRe （lib/file-utils.js / content.js ネットワーク経路ガード / page-interceptor.js）で ブロックされ結果に現れてはいけません。ストリーミングは絶対に対応しません。

  • HLS プレイリスト (playlist.m3u8) — 検出されない
  • DASH マニフェスト (manifest.mpd) — 検出されない
  • セグメント (segment.m4s) — 検出されない（avExtRe がブロック）
  • blob: 動画 — 検出されない
  • 音声直リンク (sample.mp3) — 検出されない
  • 音声直リンク (sample.wav) — 検出されない
  • 音声直リンク (sample.ogg) — 検出されない
  • 音声直リンク (sample.m4a) — 検出されない
  • 音声直リンク (sample.flac) — 検出されない
