19. 分割アーカイブ（マルチボリューム）連番

大きなアーカイブを分割した連番ファイル（HJSplit / split / 7-Zip / WinRAR / WinZip）の検出。誤検出ゼロを守る2段構え。

← トップ
期待される検出件数: 上の「検出されるべき」セクションから 合計 17 件（すべて圧縮ファイル系）。 下の「検出されてはいけない」セクションからは 0 件（＝精度ガードが効いていることの確認）。 ポップアップから Scan を実行して突き合わせてください（保存先を選べば実ファイルのダウンロードまで確認できます）。

✅ 検出されるべき（17件）

数字だけの連番（HJSplit / split）— 同一ベースの連番系列のみ昇格

  • backup.001（.001 始まり）
  • backup.002
  • backup.003
  • archive.000（.000 始まり）
  • archive.001

埋め込み拡張子を残した連番（7-Zip / 分割zip）

  • dataset.7z.001（7z として分類）
  • dataset.7z.002
  • photos.zip.001（zip として分類）
  • photos.zip.002

WinRAR 旧形式（.rar + .r00/.r01）

  • game.rar（先頭ボリューム）
  • game.r00（高信頼・単独でも検出）
  • game.r01

WinZip / zip -s 分割（.zip + .z01/.z02）

  • docs.zip（先頭ボリューム）
  • docs.z01（高信頼・単独でも検出）
  • docs.z02

WinRAR 新形式（.part1.rar / .part2.rar）

  • bundle.part1.rar（.rar 直で検出）
  • bundle.part2.rar

🚫 検出されてはいけない（精度ガードの確認・0件）

これらは意図的に 検出しません。数字だけの拡張子を無条件に拾うと ID / バージョン / 連番ページを大量に誤検出するため、 「同一ベースの連番が2つ以上・0/1始まり・同一ディレクトリ・音声/動画でない」系列だけを昇格させています。
  • lonely.001 — 単独（連番の仲間がいない）
  • gaps.001 / gaps.050 — 連続していない
  • post.1001 / post.1002 / post.1003 — 連番IDっぽいリンク（.001 始まりでない）
  • movie.mkv.001 / movie.mkv.002 — 生の動画を分割（AV除外・ストリームリッパー禁止）
  • a/vol.001 / b/vol.002 — 同名だが別ディレクトリ（無関係）
検出ロジック: lib/file-utils.js（splitVolumeHighRe / splitVolumeSeriesBases）、 popup.js の scanFunc 系列プリパス。回帰テスト: test-split-volume.cjs。
