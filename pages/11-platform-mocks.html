Salesforce Experience Cloud: .filerow + shepherd URL 合成

Lightning Community / Experience Cloud のファイル一覧では、タイトルリンクが href="javascript:void(0)" で実 URL を持ちません。拡張機能は各 .filerow から次の手がかりで credentialed な shepherd ダウンロード URL を合成します。

  • 第 1 候補（069 = ContentDocumentId）: [data-divid] 属性または a[href*="/contentdocument/069…"] から 069 で始まる 15〜18 文字の ID を抽出し、 <base>document/download/069… を組み立てます。
  • 第 2 候補（068 = ContentVersionId、フォールバック）: 069 が取れない場合、サムネイル img[src*="servlet.shepherd"] の versionId=068… から ID を回収し、 <base>version/download/068… を組み立てます。
  • <base> の決定: サムネイル src が …/sfc/servlet.shepherd/ までのパスを既に含む場合はそれを正とします （コミュニティ接頭辞が自動的に正しくなる）。無い場合は location.origin + コミュニティ接頭辞（URL の /s/ 直前のパス、例 /ass）+ /sfc/servlet.shepherd/ を使います。
  • 拡張子（typeHint）: rendition=ORIGINAL_<Ext>、 サブタイトルの中黒（•）区切り末尾トークン、または page= 付きサムネイル（→ pdf 推定）から判定し、 拡張子なしの合成 URL が nonFileUrlRe によるコミュニティページ除外を生き残るよう付与します。

このフィクスチャは https://example.my.site.com/ass/s/files のような コミュニティ接頭辞 /ass 付きの Experience Cloud ページを想定しています。 実際の file:// / localhost での閲覧では location.pathname に /s/ が無いため、 接頭辞付き <base> は サムネイル src のパス（069 行・068 行の両方に明示）から導出されます。
