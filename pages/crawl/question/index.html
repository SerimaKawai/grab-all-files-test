18-C. icon-pdf 拡張子なしリンクが解決される先（crawl/question/）

← トップ ｜ ← 18. ハブへ戻る
このページは「クロールで辿られる子ページ」ではない。 ハブ（18-crawl-nav.html）の <a class="icon-pdf" href="crawl/question/">問題PDF</a> は、 fetchAndParse のアンカーループ内で detectIconTypeHint が アイコンクラス icon-pdf を読み、その URL crawl/question/ 自体を PDF 候補（typeHint=pdf）として深度 0 で files に追加する （toshin new_kakomon_db と同型）。
つまり拡張機能は crawl/question/ を「PDF へのリンク」とみなして そのままダウンロード対象にするだけで、HTML ページとしてクロール（解析）はしない。 このため下に置いた ../../../files/sample-2.pdf は個別には検出されない。
静的ホストの注意: 本テストサイトではこのディレクトリ crawl/question/ は index.html（このページ）に解決される。 したがって crawl/question/ をダウンロードしても実体は PDF バイト列ではなく この HTML が返るため、ダウンロードの正当性は保証されない。 確認したいのは「拡張子なし URL が icon-pdf の type ヒントで 深度 0 の PDF として検出されること」であり、そこが本テストの主眼。
