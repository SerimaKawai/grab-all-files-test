18. クローラのナビ抽出（<a href> 以外）

深度 1 以上のクロールで、素の <a href> 以外のナビゲーション （javascript: / onclick / アイコンクラス付き拡張子なしリンク / サーバスクリプト .php リンク）から子ページを辿り、その先の PDF を検出できるかのテスト。

← トップ
対象コード: popup.js の fetchAndParse（クロール時のページ解析）と crawlUrl（フロンティア管理）。
  • jsUrlLiteralRe + routeJsCandidate： a[href^="javascript:"] / [onclick] の文字列から パス形のクォート済みリテラルを抽出し、ファイル形なら addFileUrl、 それ以外はクロールリンク候補として links に積む（zocbo.com 由来）。
  • crawlFileExtRe のサーバスクリプト除外 （php|phtml|asp|aspx|jsp|jspx|jspa|cfm|cgi|do|action|ashx|axd|svc|mvc）： classes.php?id=N のようなリンクは「ファイル」ではなく 「次ページへのルート」としてフロンティアに積まれる（drop されない） （yotsuyaotsuka chugaku_kakomon 由来）。
  • detectIconTypeHint + icon-pdf： 拡張子のない <a class="icon-pdf" href=".../question/">問題PDF</a> は アイコンクラスの type ヒントで PDF とみなす（toshin new_kakomon_db 由来）。
確認方法: 拡張機能でこのページを クロール深度 1 以上（ディープスキャン）で Scan してください。
深度 0（このページのみ）では、このハブ自身が直接持つファイルのみが検出されます（下記「このページ直下のファイル」）。
深度 1 で、下の各ナビ（JS / onclick / icon-pdf / .php相当）が指す子ページに到達し、 そこにある ../../files/*.pdf が検出されます。

D. サーバスクリプト .php ナビリンク

学校別ページ（classes.php?id=1）

学校別ページ（classes.html?id=1・静的ホスト用の同等品）

.php についての注意（静的ホストの制約）: classes.php?id=1 は crawlFileExtRe のサーバスクリプト除外により 「ファイル」とは扱われず、クロールフロンティアに積まれる（drop されない）のが 正しい挙動。これは php が allKnownExts に含まれるため かつては classes.php がファイル扱いされてクロールから落ち、学校ページに到達できなかった 回帰（yotsuyaotsuka）の修正点。
ただし GitHub Pages は .php を実行・配信できないため、 実機のライブ検証では classes.php?id=1 自体は 404 になり中身を解析できない。 そこで同じナビ構造を辿れるよう、classes.html?id=1 を等価の到達可能ページとして併置している。 .php リンクは「クローラが enqueue する（落とさない）こと」を文書化するための例であり、 クエリ付き .html 版で実際の到達と PDF 検出を確認できる。

期待される検出

深度 0（このページのみ）で検出されるべき:
  • ../files/sample.pdf（ハブ直下の素の a href）— 1 件のみ。
深度 0 で検出されてはならないもの:
  • A/B/C/D の子ページ内の PDF（深度 1 以上で初めて辿られる）。
深度 1（ディープスキャン）で追加検出されるべき:
  • crawl/js-target.html 経由 → ../../files/sample-1.pdf （javascript: href から jsUrlLiteralRe で抽出）。
  • crawl/onclick-target.html 経由 → ../../files/sample.docx （onclick から jsUrlLiteralRe で抽出）。docx は素の a href なので確実に検出。
  • crawl/question/（= crawl/question/index.html）経由 → ../../files/sample-2.pdf （拡張子なし URL を icon-pdf の type ヒントで PDF とみなしページとして辿る）。
  • crawl/classes.html?id=1 経由 → ../../files/sample.pdf （サーバスクリプト相当のナビが enqueue され到達。実機では .html 版で確認）。
挙動として確認したいこと（検出件数ではなく経路）:
  • javascript: / onclick 内のクォート済みパス（A・B）がフロンティアに積まれること。
  • icon-pdf の拡張子なし URL（C）が「ナビ」ではなく「PDF を持つページ」として辿られること。
  • classes.php?id=1（D）が crawlFileExtRe でファイル扱いされず enqueue されること （静的ホストでは 404 だが、enqueue されること自体が正しい。.html 版で到達を確認）。
検出されてはならないもの:
  • goPage / location.href などの関数名やパス形でない引数 （'#' 等）はファイルにもクロールリンクにもならない。
