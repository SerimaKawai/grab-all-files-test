13. 直リンク動画 / 動画・音声要素 / SVG / 短縮 data-*

← トップ
2026-06-13 改訂。v4.5.x で「直リンク動画」が検出対象に追加され、 2026-06-13 に <video src> / <source src> の動画本体も検出対象になりました（旧「poster と track のみ」を転換）。 検出ロジックは lib/file-utils.js の knownExtRe（mp4/m4v/mov/webm/avi/mkv/wmv/flv/f4v/mpeg/mpg/m2v/3gp/3g2/ogv/divx/asf/rm/rmvb/vob を収録）と、 popup.js の scanFunc addUrl / extInPath、表示ゲート shouldKeepScannedItem（scanTypeGroups の video グループ）にあります。
重要な仕様（要確認）: popup.js の deepQuery("video, audio") ループは poster（画像）・<track>（字幕）・要素自身の src/currentSrc・<source src> を収集します。 ただし media src は addUrl(src, alt, false, …)（guess=false）→ extInPath/extRe ゲート経由なので、 選択中の直リンク動画拡張子（mp4/webm/mov…）のみ通過します＝<a href="x.mp4"> と同じ境界。 <audio> の音声本体・<source> も同じく採取しますが、音声拡張子は knownExtRe・video グループのどちらにも無いため extRe ゲートで脱落＝結果に出ません（挙動は従来通り「非検出」）。 ストリーミング（HLS .m3u8 / DASH .mpd / .m4s / .m2ts / .mts）と 全音声、拡張子なし・blob/傍受メディアは extRe ゲートで脱落し、加えて avExtRe（lib/file-utils.js / content.js ネットワーク経路ガード / page-interceptor.js）でも恒久ブロックされます。
※ .ts（MPEG-TS セグメント）は否定テストに使えません。ts は TypeScript と拡張子が衝突し、web スキャングループ（デフォルト ON）の対象なので直リンクは検出されます。ストリーミングの否定確認には avExtRe が確実にブロックする .m4s を使います。
期待検出ユニーク URL: 17 件（同一ファイルを複数経路で参照しているので重複排除後の件数。2026-06-13 に sample.ogv が ② の <source> 経路で拾えるようになり 16→17）。

① 直リンク動画（v4.5.x で対応） — 検出される

<a href="x.mp4"> 形式の直リンク、または URL 末尾が動画拡張子のもの。 これらは knownExtRe にマッチし、video スキャングループ（デフォルト ON）で表示されます。 すべて検出されます。

  • 講義録画 (sample.mp4)
  • プロモ動画 (sample.mov)
  • アーカイブ動画 (sample.mkv)
  • Web 動画 (sample.webm)
  • 旧形式動画 (sample.avi)
  • Windows Media (sample.wmv)
  • Flash 動画 (sample.flv)
  • iTunes 動画 (sample.m4v)
  • MPEG 動画 (sample.mpeg)
  • モバイル動画 (sample.3gp)

② <video> 要素（src / source / track / poster）

仕様（2026-06-13 転換）: <video src> と各 <source src> も extRe ゲート経由で検出されます（直リンク動画拡張子のみ。<a href> と同じ境界）。 poster（画像）と <track>（字幕）も従来通り収集します。

  • video + 複数 source（mp4 / webm / ogv）（source も検出される） + track 字幕（検出される） + poster（検出される）:

注: sample.mp4 / sample.webm は①の直リンクで既に検出済み（②の <source> 経路は重複排除で同一URL）。 sample.ogv は直リンクが無く②の <source> でしか参照されないが、 ogv は video グループ収録のため ②経路で新たに検出されます（16→17 の +1）。

③ <audio> 要素（src / source 5 MIME） — すべて検出されない

仕様: <audio> の本体 src・<source> も②と同じく採取は試みますが、 音声拡張子（mp3/ogg/wav/m4a/flac）は knownExtRe・video グループのどちらにも無いため extInPath/extRe ゲートで脱落し、加えて avExtRe でもブロックされるため一切検出されません。 poster も付きません（poster は video 専用）。

④ SVG <image> / <use> 外部参照

外部 SVG への参照（modern href と legacy xlink:href 両方）。 <use href="#inline-sprite"> は同一ドキュメント内 fragment なので検出されません。

⑤ 画像系 data-* 短縮形（10種） — すべて検出される

popup.js の data-* 画像ローダー（addImg）が拾います。値の実拡張子で type hint を付与します。

  • data-poster
  • data-pic
  • data-photo
  • data-icon
  • data-cover
  • data-cover-image
  • data-srcset
  • data-banner
  • data-hero
  • data-avatar

⑥ メディア系 data-*（3種）

仕様: data-mp4 / data-webm は値が動画拡張子で終わるため extInPath で 検出されます（video グループ）。 data-mp3 は knownExtRe に無く avExtRe でもブロックされるため 検出されません。

  • data-mp4（検出される）
  • data-webm（検出される）
  • data-mp3（検出されない）

⑦ 検出されない動画 / 音声（avExtRe による恒久ブロック）

以下は直リンク <a href> として置いても、avExtRe （lib/file-utils.js / content.js ネットワーク経路ガード / page-interceptor.js）で ブロックされ結果に現れてはいけません。ストリーミングは絶対に対応しません。

  • HLS プレイリスト (playlist.m3u8) — 検出されない
  • DASH マニフェスト (manifest.mpd) — 検出されない
  • セグメント (segment.m4s) — 検出されない（avExtRe がブロック）
  • blob: 動画 — 検出されない
  • 音声直リンク (sample.mp3) — 検出されない
  • 音声直リンク (sample.wav) — 検出されない
  • 音声直リンク (sample.ogg) — 検出されない
  • 音声直リンク (sample.m4a) — 検出されない
  • 音声直リンク (sample.flac) — 検出されない
